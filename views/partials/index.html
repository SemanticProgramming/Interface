Welcome to Morph·D·Base Demo!

This page shows the current development of our efforts. Although we tested it thoroughly, we are expecting bug reports and will do our best to fix them. Morph·D·Base is an open web data base that has been developed to serve scientific research and education.

It provides a platform for storing the detailed documentation of all material, methods, procedures, and concepts applied, together with the specific parameters, values, techniques, and instruments used during morphological data production.

In other words, it's purpose is to provide a publicly available resource for recording and documenting morphological metadata. Moreover, it is also a repository for different types of media files that can be uploaded in order to serve as support and empirical substantiation of the results of morphological investigations.

Please refer to our wiki page for updates in the current development.

Morph·D·Base: an eScience-Compliant application for Morphology and Taxonomy

Increasing the Sustainability and Accessibility of Morphological Data through Ontology-based Standardization of Morphological and Taxonomic Descriptions and Graphical Image Content Annotations

A prototype for a Semantic Web-Content-Management-Systems

An ontology-controlled application for semantic Web-Content-Management-Systems that is self-describing and content-independent. The basic application implements a set of default features that will be available in any semantic WCMS based on the application. These features may include:

  • versioning;
  • sophisticated access-rights system;
  • management of users and user groups;
  • project workspaces;
  • entry workflow for Web-publishing, with distinct set of states that indicate the publishing progress of an entry;
  • provenance tracking;
  • dynamic forms: input forms can be customized by individual users or by project administrators;
  • efficient grid editing of bulks of data entries;
  • SPARQL-driven facetted search;
  • application GUI for adapting and modifying pre-defined data view templates and data base workflows and for describing new entry forms and workflows;
  • Linked-Open-Data ready -> inter-linking of data entries possible. Even between data stored on different semantic WCMS that are based on this ontology-controlled application;
  • general Granularity Framework for defining Views and Perspectives on the data that facilitate data organization and exploring the application’s contents from different point of views.
